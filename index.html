Project 2: Responsive Redesign.

The problems with the original site

The original website has many problems, both in usability and accessibilty. First of all, it is very boring, with no structure or color. This makes the page hard to navigate. It also creates problems if somebody is looking for a specific piece of information, since the original page is not organized but rather just a clutter of links. This creates problems for usability, learnability and memorability, since as things are updated, the website changes completely. Accessibility problems found by WAVE include heading structure, no page layout, and very small text. All of these findings are true, and can impede accessibility; small text can be hard to read, and the lack of general layout can impede users who rely on things like screen readers to know where they are on the page. WAVE noted that the page was made up entirely of unordered lists, further reinforcing the lack of structure.

Hi-Fi Prototypes

These hi-fi prototypes were created using Figma, which will help us build the website. These hi-fi prototypes greatly spread out the links, so the page is not as cluttered. Additionally, there are now a dedicated sections for news releases and letters, so that the website is better organized to fit the users' conceptual models. The structure both helps accessibility and usability, while preserving the important links on the front page; other info can be found by scrolling.
